# Архитектура проекта pdf2jpg_service

## Компоненты

- **GUI (PyQt5)** — отвечает за взаимодействие с пользователем, выбор файлов и папки вывода, отображение прогресса.
- **Converter (PyMuPDF + Pillow)** — рендерит страницы PDF в изображения и сохраняет их в формате JPEG.
- **Logger** — централизованная настройка логирования (консоль + файл).
- **Tests** — unit-тесты для проверки корректности конвертации.

## Поток данных

1. Пользователь выбирает PDF и папку вывода в GUI.
2. GUI создаёт фоновую задачу (QRunnable) и запускает её в ThreadPool.
3. Фоновая задача вызывает `convert_pdf_to_jpg`, которая:
   - Открывает PDF через PyMuPDF.
   - Рендерит каждую страницу в pixmap.
   - Конвертирует pixmap в PIL.Image.
   - Сохраняет изображения в указанную папку.
4. GUI получает сигнал об окончании и показывает сообщение пользователю.

## Расширения и улучшения

- Параллельный рендер страниц (ThreadPoolExecutor) для больших PDF.
- Поддержка других форматов (PNG, WEBP).
- CLI-режим для пакетной обработки.
- Интеграция с CI (pytest, mypy, flake8).
